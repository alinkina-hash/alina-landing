## 3. Контент лендинга

### Секции и их статус
| Секция | Статус | Заметки |
|---|---|---|
| Герой (слоган + lead) | ✅ готово | Финальный текст вписан |
| Trust-strip | ✅ готово | Финальный текст вписан |
| Обо мне | ⬜ заглушка | Написать: путь, образование, сертификация, взгляд |
| Метод | ✅ готово | Текст вписан (пересказ сути, не копипаст) |
| Аудитория (5 портретов) | ✅ готово | Все 5 портретов вписаны |
| С чем приходят (сферы) | ✅ готово | 10 запросов-болей |
| Запись | ✅ готово | Мок-календарь, правило 24ч |
| Стоимость | ✅ готово | «Цена по запросу», список что входит |
| Отзывы | ⬜ заглушки | Заменить реальными отзывами клиентов |
| Контакты | ✅ готово | Telegram + Instagram (ссылки подставить) |

### 5 портретов аудитории
1. **Активная женщина в тупике** — есть всё, но ощущение, что застряли и ходите по кругу.
2. **Кризис перехода** — развод, переезд, новая роль — почва ушла из-под ног.
3. **Ищущий и осознанный** — уже многое прошли, хотите идти глубже, к сути.
4. **Предприниматель и топ-менеджер** — внешние результаты есть, внутреннего ресурса не хватает.
5. **Женщина в отношениях** — повторяются сценарии, не складывается близость.

### 10 запросов-болей («С чем приходят»)
1. Одни и те же ситуации повторяются в моей жизни
2. Меня триггерят определённые люди — и я не понимаю почему
3. Повторяю один сценарий в отношениях
4. Чувствую тревогу, вину или обиду без явной причины
5. Не могу реализоваться, упираюсь в потолок
6. Сложные отношения с родителями и родом
7. Хочу понять своё предназначение и куда двигаться
8. Самооценка и ощущение собственной ценности
9. Эмоциональные качели, выгорание
10. Не умею выставлять границы

### Описание метода (финальный текст)
> Вселенская терапия — это структурный метод с чёткой схемой работы. Вместо многолетних «раскопок» и бесконечных историй мы сразу ищем причину: почему это происходит именно с вами и что за ней стоит на самом деле. Широкий инструментарий — подход подбирается под каждого человека и его запрос.

Три шага:
- **01 Сразу в суть** — не тратим время на длинные истории, ищем что стоит за ситуацией на самом деле.
- **02 Системная работа** — чёткая схема для любого запроса: отношения, самооценка, тело, реализация, семейные сценарии. Инструментарий подбирается индивидуально.
- **03 Высокая скорость** — сложные жизненные вопросы решаются за 1–5 сессий. Вы уходите с ясностью и конкретным результатом.

## 4. PRD: продуктовые требования

### Суть продукта
Одно личное веб-приложение для ведения практики. Лендинг и back-office — **один тул**. Никакого SaaS. Бюджет = **$0**.

### Роли
- **Я (специалист)** — вожу клиентов, сессии, заметки, ДЗ, вижу аналитику.
- **Клиент** — записывается, получает напоминания, видит свои сессии/заметки/ДЗ, оставляет фидбэк. Клиенты из любой точки мира.

### Модули MVP
| Модуль | Суть |
|---|---|
| **M0 Лендинг** | Публичная страница приложения (все секции выше) |
| **M1 Бронирование** | Cal.com: свободные + занятые слоты (занятые без данных клиента), онлайн/офлайн, таймзона клиента, отмена за 24ч |
| **M2 Оплата** | Вручную вне приложения; приложение фиксирует только статус |
| **M3 CRM** | Карточка клиента: контакты, источник, запрос, теги, статус |
| **M4 Сессии** | Дата/формат/статус + **приватные заметки** (только я) + **общие заметки** (видны клиенту) + ДЗ с трекингом |
| **M5 Кабинет клиента** | Google-Doc на клиента, доступ по ссылке (magic-link), без пароля |
| **M6 Напоминания** | Email: подтверждение брони, напоминание за 24ч и 1–2ч, запрос фидбэка |
| **M7 Фидбэк** | Оценка 1–5 + комментарий; публикация на лендинге — на моё усмотрение |
| **M8 Аналитика** | Число сессий, средний рейтинг, загрузка, сводка оплат |

### Позже (после MVP)
- **M9 AI-ассист** — черновики резюме сессий, подсказки по ДЗ, подготовка к сессии (Claude API)
- Расширенная аналитика (динамика по клиенту, источники, retention)

### Вне объёма
Онлайн-оплата, видеозвонки внутри приложения, мобильные приложения, групповые сессии, SaaS для других специалистов.

## 5. Технический стек ($0, no-code)

| Что | Инструмент | Лимиты / заметки |
|---|---|---|
| Лендинг (хостинг) | **Vercel (Free)** + GitHub | Бесплатно, авто-деплой при push |
| Лендинг (файл) | `index.html` | Текущий файл: `landing-alina-shvedova.html` |
| Бронирование + напоминания | **Cal.com (Free)** | Безлим. слотов, вебхуки, авто-напоминания |
| «Занятость» на лендинге | **Google Calendar** («свободно/занято») | «Занято» без деталей клиента |
| База клиентов (CRM + сессии) | **Airtable (Free)** | ~1 000 записей/база |
| Кабинет клиента | **Google-Doc на клиента** | $0, безлимит клиентов, вручную |
| Синхронизация | **Make.com (Free)** | ~1 000 операций/мес |
| Фидбэк | **Google Forms** → Airtable | Бесплатно |
| Email | **Brevo (Free)** | ~300 писем/день |
| AI-ассист (позже) | **Claude API** | Копейки за вызов |

### Деплой
- Репозиторий: GitHub → `alina-landing`
- Хостинг: Vercel, подключён к репозиторию
- Файл: `index.html` в корне репозитория
- Обновление: push в GitHub → Vercel автоматически переопубликовывает
